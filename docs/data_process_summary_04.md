## Implemented Code

### Step 3: Split Builder

Implemented `utils/09_build_data_splits.py`.

The script reads each task under `data/training_ready/<dataset_group>/tasks/<task_name>/`, builds pairings from the task `feature_table`, and writes split artifacts under `data/training_ready/<dataset_group>/splits/<task_name>/`.

Generated artifacts include:

- `set_info.pkl`
- `row_to_set_index.pkl`
- `set_to_grouping.pkl`
- `train_indices_<strategy>.pkl`
- `valid_indices_<strategy>.pkl`
- `test_indices_<strategy>.pkl`
- compatibility aliases `val_indices_<strategy>.pkl`
- per-split `*_set_info_<strategy>.pkl`
- per-task `split_manifest.json`
- overall `data/training_ready/split_build_manifest.json`

Split row indices are feature-table row indices. Only rows satisfying this anchor rule are split:

```text
not is_control and source_row_role == self and feature_membership == primary
```

Control rows are not split as anchors. They are linked through `set_info.pkl` and sampled by the training dataset.

Implemented split policies:

- PTV3 main single drug: `random`, `cell`, `cell_type`, `pert_stratified`, 5-fold `pert_id`, 5-fold `cell`, 5-fold `cell_type`, 5-fold `pert_stratified`, and `all_train_subset_test`.
- PTV3 main double drug: 5-fold `pert_id` and `all_train_subset_test`.
- PTV3 extra single-drug and extra double-drug tasks: `test_only`.
- PTV1 main AIVC: `fixed_experiment_type`, 5-fold `pert_id`, and `all_train_subset_test`.
- PTV1 extra single drug: `test_only`.

### Step 4: Dataset, Models, Training, and Inference

Implemented:

- `dataset/__init__.py`
- `dataset/training_ready_dataset.py`
- `model/__init__.py`
- `model/training_ready_models.py`
- `model/training_ready_lightning.py`
- `train.py`
- `infer.py`

The new dataset reads:

- `feature_table.parquet` / `.pkl` / `.csv`
- `feature_expression_matrix.npy`
- `feature_ordered_protein_index.json`
- `feature_sample_ids.json`
- `global_meta.json`
- split pickle files generated by Step 3

The dataset returns one paired sample per item:

```python
{
    "control": {...},
    "perturb": {...},
}
```

After DataLoader collation, models use the same double-drug perturbation contract without a group axis. `pert_id` contains both drugs:

- baseline model: shape `[batch_size, 2, drug_embedding_dim]`
- graph models: shape `[batch_size, 2]`

For single-drug rows, `pert_index2` is expected to be the metadata `no` index.

Current implemented model registry names:

- `attention_v10_hetero_cls_ee`
- `baseline_emb_v3`

`attention_v10_hetero_cls_ee` is the consolidated PDI hetero graph model. `--use-target` controls target protein tokens and `--target-protein-fusion-model gate` controls the gene/protein gate. Current PDI artifacts are stored as `[n_drug, n_protein]`; the model code transposes this by default before building the legacy PyG `protein -> drug` graph.

Training writes a run manifest under the checkpoint directory and supports:

- training-ready root selection
- dataset group/task/split selection
- selected model selection
- double-drug input by default
- PDI graph model loading
- configurable label keys
- finite-mask MSE loss over expression labels
- BCE losses for response and synergy labels
- optional dry-run batches
- dry-run batches do not write checkpoint manifests; actual training writes the manifest before `trainer.fit`

Inference writes:

- `predictions.parquet` if parquet support is available, otherwise CSV fallback
- `metrics.json`
- optional `expression_pred.npy`
- `run_manifest.json`

## Implementation Details and Assumptions to Review

1. `double_drug` pure `pert_id` 5-fold split is interpreted as the ordered pair `pert_id1 + pert_id2`.
   - Reason: the current feature table has two perturbation columns.
   - Alternative to decide later: unordered pair, first-drug-only, second-drug-only, or any-drug holdout.

2. PTV3 extra single-drug tasks are treated as `test_only`.
   - Reason: Step 3 explicitly names `extra_guomics`, `nc`, and `nature` as test-only, while Step 4 says inference should target `extra_singledrug` and `extra_doubledrug`.
   - This is recorded because Step 3 did not explicitly list every extra single-drug task name.

3. `all_train_subset_test` allows train/test overlap by design.
   - The script still keeps valid/test disjoint.
   - The manifest records overlap counts and `allow_train_test_overlap`.

4. The PTV1 fixed split parses `data/rawdata/ptv1/experiment_type_list` directly in Step 3.
   - PTV1 also has a separate `random` split.
   - `split_manifest.json` records how many valid anchors are unmatched by the fixed experiment-type lists.

5. The selected model classes now use legacy-style internals rather than the previous compact context model.
   - Active model choices are now only `attention_v10_hetero_cls_ee` and `baseline_emb_v3`.
   - `attention_v10_hetero_cls_ee` uses the legacy PDI graph, token/CLS Transformer path, and double-drug response/synergy heads.
   - Target protein tokens are controlled by `--use-target`.
   - Gene/protein gate fusion is controlled by `--target-protein-fusion-model gate`.
   - All active models keep the current double-drug input/output contract.

6. `baseline_emb_v3` now follows the historical baseline embedding + fusion MLP pattern.
   - It requires `--emb-dataset-path`.
   - The embedding dataset row order must match the current feature table row order.
   - A synergy head is added so it can run under the current double-drug trainer.

7. Graph models use PDI only.
   - This follows Step 4.
   - PPI and DDI are not consumed by the Step 4 graph model code.

8. PDI artifact orientation is `[drug, protein]`.
   - This follows the current matrix builder output.
   - Legacy `create_pdi_only_graph` expects `[protein, drug]`.
   - The current model code defaults `--pdi-input-orientation drug_by_protein` and transposes before building `protein -> drug` and reverse edges.

9. `batch_cov_list` defaults to `machineID_new Cell_plate Cell cell_type batch pert_time`.
   - `batch` is not included because the current training-ready tables do not expose a `batch_index` column.
   - `pert_dose1` and `pert_dose2` are supported if passed explicitly and if their index columns exist.

10. `effective_key1` is inferred automatically.
    - Extra single-drug and `ptv1_extra_singledrug` tasks use `PRISM2nd_label_total`.
    - Other tasks use `PRISM1st_label_total`.
    - `effective_key2` defaults to `synergy`.

11. Response label encoding treats `sensitive/responsive/yes/1/true` as positive and `non-responsive/nonresponsive/no/0/false` as negative.

12. Synergy label encoding treats `syn/synergy/synergistic/yes/1/true` as positive and `non-syn/nonsyn/non-synergy/non_synergy/no/0/false` as negative.

13. Expression MSE follows the legacy NaN-mask behavior for perturb expression labels.
    - Control expression inputs are no longer sanitized in the dataset.
    - Control NaNs are passed into the legacy-style `ValueEmbedding`, which maps NaNs through its learned `nan_embedding`.
    - Validation/test now log the legacy-style expression metric suite plus both classification heads.

14. Training uses validation for checkpoint selection and tests after fit from the best validation checkpoint.
    - `train.py` now supports `--skip-test` and `--limit-test-batches`.
    - The run manifest records `best_model_path`, `best_model_score`, and `test_checkpoint_path`.

15. `infer.py` writes prediction probabilities for task 1 and task 2, plus available raw label columns.
    - For unlabeled or not-applicable labels, AUROC/AUPRC can be `NaN`.
    - This was observed for task 1 on `ptv3_extra_doubledrug_guomics`, where task 2 synergy metrics were valid.
    - When `--save-expression-pred` is enabled, inference also writes full `legacy_validation_metrics` using the validation/test expression metric suite.

## Legacy Behavior Review

Detailed reviews are recorded in:

- `data/review_summary/2026-05-07_1622_legacy_behavior_gap_review.md`
- `data/review_summary/2026-05-07_1643_legacy_model_port_and_entrypoint_review.md`
- `data/review_summary/2026-05-07_1709_training_metrics_nan_forward_review.md`
- `data/review_summary/2026-05-07_1733_legacy_trainer_options_model_consolidation_infer_metrics.md`

Current conclusion:

1. Dataset sampling is close to legacy, but storage and feature encoding differ because the new pipeline reads training-ready feature tables plus `.npy` expression matrices.
2. Trainer metrics now restore the legacy-style validation/test expression metrics and dual classification metrics.
3. `train.py` / `infer.py` are unified around training-ready artifacts; legacy scripts used separate single/double entry points and separate inference split folders.
4. The previous compact model blocker has been addressed by porting legacy-style token/CLS internals and the legacy PyG PDI graph path.
5. Trainer options now include focal loss, positive weights, `adamw_fused`, `step`, `cosine_warmup`, and `UnfreezeCallback`.
6. Practical training risk remains: current PTV3 feature matrices contain about 11k protein tokens; top-k protein selection was explicitly skipped by the user, so full PTV3 training still requires enough GPU memory for full attention.
