# 2026-05-09 14:28 HKT Inference Manifest Strictness Review

## Scope

- Re-audit external inference safety after the training-entrypoint guard.
- Focus on whether checkpoint config validation can be bypassed by missing or incomplete manifests.

## Loopholes Found

- `infer.py` skipped checkpoint config validation when `run_manifest.json` was missing next to the checkpoint.
- `infer.py` ignored manifest keys that were missing from the checkpoint manifest, so old or incomplete manifests could bypass newly required checks such as `task_head`.

## Fixes

- `infer.py` now requires `run_manifest.json` by default for checkpoint-backed inference.
- Missing manifests only pass with explicit `--allow-missing-checkpoint-manifest`.
- Missing required config keys now count as checkpoint config mismatches.
- For non-graph/axis-fixed models, missing `ordered_protein_index_path` now counts as a mismatch; if present, the protein axis must exactly match.
