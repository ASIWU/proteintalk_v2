# 2026-05-14 15:41 HKT Extra Raw Control Source Restriction Review

## Scope

- Reviewed user-reported issue that exported `control` values included `control_*` ids.
- Rechecked the source of those ids and updated `utils/10_export_extra_raw_with_controls.py`.

## Finding

- `control_*` values were not generated by the export script.
- They came from existing `ptv3_main_doubledrug` control rows / references because the previous export reused stage-2 processed controls, whose historical matching pool included main double-drug controls.
- This was inconsistent with the intended external export contract: control ids should come from original `ptv3_main_singledrug` or `ptv3_extra_baseline`.

## Fix

- Stage-2 `processed.csv` is now used only to decide whether a raw row survived filtering.
- Exported `control` is rematched from a restricted control pool:
  - `ptv3_main_singledrug` rows where `control == sample_id`;
  - all `ptv3_extra_baseline` sample ids.
- The exported match audit columns are recomputed from that restricted pool.
